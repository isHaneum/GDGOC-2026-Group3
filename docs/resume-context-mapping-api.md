# Resume Context Mapping API

## Purpose

The Resume Context Mapping API converts a Korean or Japanese resume into recruiter-facing context for the opposite locale. It is not a literal translation endpoint. The goal is to preserve facts while explaining resume conventions, self-introduction structure, timing gaps, and culture-specific assumptions that may otherwise be misread by recruiters in Korea or Japan.

The first backend implementation supports Korean-to-Japanese and Japanese-to-Korean mapping only. Frontend rendering is intentionally out of scope for this phase.

## Endpoint

```http
POST /api/map-resume-context
Content-Type: application/json
```

Runtime: `nodejs`

The route is implemented in `app/api/map-resume-context/route.ts` and delegates to `mapResumeContextPayload` in `server/services/apiHandlers.ts`.

## Request Interface

```ts
type ResumeContextLocale = "ko" | "ja";

interface ResumeContextMappingContent {
  name: string;
  content: string;
}

interface ResumeContextMappingRequest {
  targetLocale: ResumeContextLocale;
  contents: ResumeContextMappingContent[];
}
```

Example:

```json
{
  "targetLocale": "ja",
  "contents": [
    {
      "name": "Name",
      "content": "박지훈"
    },
    {
      "name": "Education and Military Service Timeline",
      "content": "2014년 3월 컴퓨터공학과에 입학했고 2016년부터 2018년까지 육군 통신병으로 복무했습니다."
    }
  ]
}
```

## Validation

Invalid input returns `400`.

Rules:

- `targetLocale` must be `"ko"` or `"ja"`.
- `contents` must be an array.
- `contents` length must be between 1 and 30.
- Every item must have non-empty string `name` and `content`.
- Each `content` value must be 4,000 characters or less.
- Combined `content` length must be 40,000 characters or less.
- Malformed JSON returns `400` with `Request body must be valid JSON.`

Validation is implemented by `validateResumeContextMappingRequest` in `server/services/resumeContextMapper.ts`.

## Success Response Interface

```ts
type ResumeDetectedLocale = "ko" | "ja" | "mixed" | "unknown";
type ResumeContextConfidence = "high" | "medium" | "low";

interface ResumeContextNote {
  note: string;
  confidence: ResumeContextConfidence;
  basis?: string;
}

interface ResumeContextMappedItem {
  name: string;
  mappedName: string;
  originalContent: string;
  mappedContent: string;
  detectedSourceLocale: ResumeDetectedLocale;
  contextNotes: ResumeContextNote[];
}

interface ResumeContextMappingResult {
  id: string;
  createdAt: string;
  targetLocale: "ko" | "ja";
  detectedSourceLocale: ResumeDetectedLocale;
  items: ResumeContextMappedItem[];
}
```

Notes:

- `id` is generated with `crypto.randomUUID()`.
- `createdAt` is an ISO timestamp generated by the server.
- `mappedName` is the field or question name adapted for the target resume context.
- `mappedContent` is the content adapted for the target language and recruiting context.
- `contextNotes` must explain factual context and cultural inference without presenting inference as personal fact.
- `contextNotes.note`, `contextNotes.basis`, and mapped text are expected in the requested `targetLocale`.

## Failure Behavior

The API returns `503` when the AI mapping service cannot produce a valid result.

Current `503` cases:

- `GEMINI_API_KEY` is missing.
- Gemini request fails.
- Gemini returns malformed JSON.
- Gemini JSON does not normalize into the expected mapping result shape.

Failed requests and partial outputs are not persisted.

## Gemini Contract

The prompt lives in:

```text
server/prompts/resume-context-mapping.md
```

The prompt instructs Gemini to:

- perform cultural and contextual mapping, not literal translation;
- preserve factual accuracy;
- map both field/question names and field content;
- write explanation fields in `targetLocale`;
- mark common cultural inference with confidence;
- avoid treating inference as confirmed personal fact;
- avoid protected-class judgments and final hiring decisions.

The Gemini response schema is defined as `resumeContextMappingSchema` in:

```text
server/services/schemas.ts
```

The service uses the existing `generateGeminiJson` wrapper and passes the schema through the shared `responseJsonSchema` pattern.

## Persistence

Successful mappings are appended to:

```text
server/data/resumeContextMappings.json
```

Persisted record shape:

```ts
interface ResumeContextMappingRecord {
  id: string;
  createdAt: string;
  request: ResumeContextMappingRequest;
  response: ResumeContextMappingResult;
}
```

The API has no list or read endpoint in this phase.

## Implementation Files

- `shared/types.ts`: shared request, response, note, item, and persisted record types.
- `server/services/resumeContextMapper.ts`: validation, prompt loading, Gemini call, response normalization, and persistence.
- `server/services/schemas.ts`: Gemini JSON response schema.
- `server/services/apiHandlers.ts`: API handler wrapper and HTTP error mapping.
- `app/api/map-resume-context/route.ts`: Next.js route.
- `src/api/client.ts`: client-side integration wrapper for later frontend handoff.
- `server/prompts/resume-context-mapping.md`: editable system prompt.

## Local Verification

Run static verification:

```bash
npm run typecheck
```

Run the local server:

```bash
npm run dev
```

Smoke test with a Korean resume targeting Japanese or a Japanese resume targeting Korean:

```bash
POST http://localhost:5173/api/map-resume-context
```

Confirm that:

- valid requests return a structured mapping result;
- invalid bodies return `400`;
- AI service failures return `503`;
- successful calls append exactly one record to `server/data/resumeContextMappings.json`;
- failed calls do not append records.
